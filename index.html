Everything you need, nothing you don't

HugoKit gives you a focused set of tools for managing Hugo sites — designed to get out of your way.

Edit in your editor — see it live in the browser

Keep writing in VS Code, Nova, or whichever editor you love. HugoKit runs the Hugo dev server in the background and pushes every change to your browser the instant you hit save — no refresh, no reload, no Terminal.

Markdown edits, template tweaks, CSS changes, image swaps — they all appear in the preview in milliseconds. Your scroll position stays put. Your focused input stays focused.

  • Instant hot reload — edits in any editor reflect in the browser on save
  • One-click start/stop/restart — no more hugo server in Terminal
  • Live log streaming with structured, color-coded output
  • Toggle drafts, future content, and expired pages on/off without restarting
  • Custom port selection — run multiple sites side by side
  • Auto-restart on config changes so you never miss an update
  • Clickable localhost URL opens your browser instantly

Content analytics

Get a bird's-eye view of everything in your Hugo site. HugoKit scans your content directory and presents key stats in a clean dashboard — no manual counting.

  • Published, draft, future, and expired page counts at a glance
  • Section-by-section breakdown of your content tree
  • Front matter health check — flags missing titles, dates, or draft fields
  • Quick-filter to surface only drafts or only scheduled posts

Visual config editor

Hugo's config file is powerful but can be intimidating. HugoKit presents it as a structured form with labels, descriptions, and inline validation — while keeping the raw file just one toggle away.

  • Supports toml, yaml, and json config formats
  • Toggle between visual form and raw text mode anytime
  • Grouped settings: general, build, server, markup, and more
  • Changes write directly to your config file — no abstraction layer

Publish anywhere

Publish your Hugo site to any hosting platform without leaving HugoKit. A step-by-step wizard guides you through setup, and subsequent publishes are one click.

  • GitHub Pages via OAuth — sets up Actions workflow automatically
  • FTP / SFTP with smart sync and macOS Keychain credential storage
  • Netlify, Vercel, and Cloudflare Pages build triggers
  • Local export to any folder for manual upload or staging
  • Publish history with timestamps and status tracking

Preflight checks

Every publish runs through a series of validation steps before anything goes live. HugoKit catches the mistakes that are easy to miss — and fixes many of them automatically.

  • Hugo build validation with detailed error output
  • Config structure check to catch syntax errors early
  • Subpath detection in templates and JavaScript files
  • Auto-fix with inline diff preview — review before applying
  • Static file validation to catch missing assets

Multi-site management

If you maintain more than one Hugo site, HugoKit keeps everything organized in a single compact window. Each site is fully isolated — its own server, config, and publish target.

  • Add sites by drag-and-drop or folder picker
  • Per-site server state, port, and log history
  • Quick-switch between sites from the sidebar or menu bar
  • Independent publish targets per site
  • Status indicators show which sites have running servers

What is Hugo?

Hugo is one of the world's most popular open-source static site generators. Write content in Markdown, apply a theme, and ship a complete website — in milliseconds.

  • Blazing fast

    Builds thousands of pages in under a second. Written in Go and optimized for speed.

  • Single binary, zero dependencies

    No runtime, no database. Just one file that turns your content into a full static site.

  • Free and open source

    MIT-licensed, backed by a massive community, and trusted by the world's biggest sites.

  • SEO-friendly by default

    Static HTML means clean URLs, instant page loads, and search engines that love your site.

HugoKit is a native macOS app that wraps Hugo in a friendly interface — so you can manage sites, preview themes, and publish without ever opening Terminal.
