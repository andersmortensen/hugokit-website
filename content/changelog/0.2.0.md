HugoKit no longer changes anything without a diff – and everything it changes can be undone.

## New

- Snapshots: before HugoKit writes to any file in your site – a preflight fix, a config save, a content edit, a theme switch – it keeps a copy of the file as it was. If the copy can't be taken, the write doesn't happen
- Every change is recorded as a per-file diff, captured at write time – read it in the new Snapshots sheet (in the site's ⋯ menu), or copy it out
- Undo works on two levels: restore a whole operation, or a single file – and restoring snapshots first, so an undo can itself be undone
- Per-site snapshot settings: turn the layer off for a site, or set how many versions of each file to keep
- Publish to All Targets – one button on the Deploy page when a site has two or more active destinations
- Each deploy target now shows its own status and history, not just the site as a whole
- Target health: after a successful publish HugoKit probes the live URL, and Check if Live in the target's ⋯ menu runs the probe on demand
- Deprecation inbox: Hugo's deprecation notices are collected into one list on the dashboard – each notice once, with a count of how often it fired

## Improved

- The Starter template grew up: an archive, a colophon, sidenotes, a timeline archive view, a ⌘K search palette, a sharing image drawn per page and the whole site as llms.txt
- The window has been redesigned: one warm surface with the sidebar and content as floating panes

## Fixed

- FTP/SFTP publish no longer skips dotfiles – .htaccess, .well-known and .nojekyll upload like everything else; only .DS_Store junk is filtered
- FTP smart sync compares files correctly when the project path goes through a symlink

## Notes

- Requires macOS 26 (Tahoe) or later
